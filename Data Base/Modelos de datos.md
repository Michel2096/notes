
Un **modelo de datos** es la forma en que **estructuras, organizas y representas la información** dentro de una base de datos.

1. Qué datos se van a guardar
 2. Cómo están organizados
3. Cómo se relacionan entre sí

![[Pasted image 20260326122847.png]]

# Qué define un modelo de datos?

Un modelo de datos especifica:

- **Estructura** → cómo se organizan los datos
- **Relaciones** → cómo se conectan
- **Restricciones** → reglas que deben cumplir

### Ejemplo

```
Alumno
- id
- nombre

Curso
- id
- nombre

Relación:
Alumno — cursa — Curso
```

# Tipos principales (resumen)

- **Modelo Entidad–Relación (ER)** → diseño conceptual
- **Modelo Relacional** → tablas en bases de datos
- **Modelo jerárquico** → estructura en árbol
- **Modelo en red** → relaciones complejas
- **Modelo NoSQL** → estructuras flexibles (JSON, grafos, etc.)
