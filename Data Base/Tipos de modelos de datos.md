Es un modelo **visual (dibujos)** que se usa para diseñar la base de datos.

🔹 Usa:

- **Rectángulos** → entidades (Alumno, Curso)
- **Óvalos** → atributos (nombre, edad)
- **Rombos** → relaciones (inscribe, compra)

🔹 Ejemplo:

- Alumno — _se inscribe en_ — Materia

👉 Sirve para **planear** cómo será la base de datos antes de crearla.

Organiza los datos como un **árbol (padre → hijo)**.

🔹 Características:

- Un padre puede tener varios hijos
- Cada hijo tiene **un solo padre**

🔹 Ejemplo:

- Empresa  
    └── Departamento  
    └── Empleado

👉 Es simple, pero **limitado** (no permite relaciones complejas).

Es como el jerárquico pero **más flexible**.

🔹 Características:

- Un nodo puede tener **varios padres**
- Permite relaciones más complejas

🔹 Ejemplo:

- Un alumno puede estar en varias materias
- Una materia puede tener varios alumnos

👉 Es más potente que el jerárquico, pero más difícil de manejar.

## Resumen 


|   **Modelo**   |  **Forma**  |         **Uso**          |
| :--------: | :-----: | :------------------: |
|     ER     | Dibujos |       Diseñar        |
| Relacional | Tablas  | Usar en la vida real |
| Jerárquico |  Árbol  | Estructuras simples  |
|   En red   |  Grafo  | Relaciones complejas |


> [!NOTE] Nota
> El modelo relacional y el modelo ER, son las mismas solo que una se representa de forma grafica y la otra en el modelo de la base de datos
